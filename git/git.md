# Git
Git - распределённая система контроля версий (version control system, VCS), которая позволяет в компактном виде хранить историю версий отслеживаемых проектов, переключаться между различными версиями и откатывать изменения.
Git в совокупности с сервисами для хостинга проектов (GitHub, GitLab, BitBucket etc.) представляет незаменимый инструмент для командной разработки.
Логика работы систем контроля версий состоит в том, что вместо хранения различных версий одного проекта в виде набора файлов и директорий для каждой версии, обеспечивается хранение последней версии проекта и истории его изменений (коммитов).
Это позволяет значительно минимизировать нагрузку на диск. Помимо этого, Git предоставляет удобный функционал для совместной разработки. Каждый разработчик может вести разработку в собственной ветке, не мешая другим.
Все эти особенности делают Git незаменимым инструментом. Его изучение может быть полезно не только разработчикам ПО, но также и нетехническим специалистам.


# Основные команды Git
1. Создание локального репозитория
$ git init
Инициализирует локальный репозиторий в текущей директории. Для инициализации репозитория не в текущей директории необходимо явно указать путь к директории проекта.
$ git clone [url]
Загружает по сети удалённый репозиторий в текущую директорию.

2. Конфигурация Git
$ git config -l
Просмотр текущей конфигурации git
$ git config --global user.name "Имя пользователя"
Добавляет имя пользователя в конфигурацию
$ git config --global user.email "example@email.com"
Добавляет почту пользователя в конфигурацию

3. Работа с изменениями
$ git status
Показывает состояние файлов в рабочей директории (какие файлы были изменены и ожидают индексации, какие проиндексированы и ожидают коммита). Дополнительно выводятся подсказки.
$ git add [file]
Добавляет файл в индекс для последующего коммита. Для добавления всех файлов в текущей директории можно указать . вместо имени файла.
$ git rm [file]
Удаляет файл из индекса и рабочей директории.
$ git commit -m "Commit description"
Коммит изменений. Используя -m можно описать для удобства закоммиченные изменения. Сохраняет слепок данных добавленных в индекс (командой git add) во внутренней базе, а затем сдвигает указатель текущей ветки на этот слепок.
$ git reset
Откат изменений.
